# Usage Guide

## 対象ユーザー

このガイドは `Folder Encryptor V5` を初めて使う人向けです。

概要だけを知りたい場合は README から読み始めてください。正式に使う予定がある場合は、このページ全体を読むことをおすすめします。

## 利用可能な版

現在は以下の 2 種類を提供しています。

- インストーラー版: `dist/FolderEncryptor_Dino_SmartSetup.exe`
- ポータブル版: `dist/FolderEncryptor_V5_Dino.exe`

通常はインストーラー版がおすすめです。すぐに試したい場合やインストールしたくない場合はポータブル版を使えます。

## 画面説明

メイン画面の主な項目:

- `Target Folder / 目标文件夹`: 暗号化または復号したいフォルダーを選択
- `Browse`: フォルダー選択ダイアログを開く
- `Master Password / 安全密码`: 自分でパスワードを入力
- `随机生成`: 強力なパスワードを自動生成
- `隐形加密 (Encrypt)`: 暗号化を実行
- `一键还原 (Decrypt)`: 復号を実行

## ランダムパスワード生成

手動でパスワードを設定したくない場合は `随机生成` をクリックしてください。

プログラムは強力なパスワードを生成し、忘れないようにその内容をデスクトップ上のバックアップ用テキストファイルにも保存します。

このバックアップファイルの特徴:

- デスクトップに txt ファイルが生成される
- `随机生成` を押すたびに新しい記録が追記される
- 以前の内容は上書きされない

例:

```text
【V5 专属加密密码备份】
密码: euZi^oahxuKq2Zm6

【V5 专属加密密码备份】
密码: JSE7CWTg54#CSH!Q
```

処理完了後は、そのバックアップファイルをより安全な場所へ移動することをおすすめします。

## 使い始める前に

最初に使う前に、重要ではないファイルを入れたテスト用フォルダーを用意し、一度暗号化と復号の流れを通して試してください。

その理由:

- 画面と操作手順に慣れられる
- 現在の PC 環境で正常に動作するか確認できる
- 最初から重要データを扱うリスクを避けられる

## フォルダー暗号化

推奨手順:

1. プログラムを起動
2. `Target Folder / 目标文件夹` で暗号化したいフォルダーを選択
3. `Master Password / 安全密码` にパスワードを入力、または `随机生成` をクリック
4. 選択したパスを確認
5. `隐形加密 (Encrypt)` をクリック
6. 処理完了まで待機
7. 結果フォルダーを開いて手動で確認

## フォルダー復号

推奨手順:

1. プログラムを起動
2. このツールで処理済みのフォルダーを選択
3. 元の暗号化パスワードを入力
4. `一键还原 (Decrypt)` をクリック
5. 復元後のファイルが正常に開けるか確認

## 注意事項

### 1. パスワード管理

- パスワードはファイル復元の鍵です
- 必ず安全に保管してください
- ランダムパスワードを使う場合はデスクトップ上のバックアップファイルも保管してください

### 2. 操作前にバックアップ

- 重要データを暗号化する前に追加バックアップを作成してください
- 特に初回の本番利用では唯一の原本を直接扱わないことをおすすめします

### 3. 小規模テストを先に行う

- まずは小さいフォルダー 1 つか 2 つで試す
- 安定していることを確認してから重要なデータへ進む

### 4. 環境に関する注意

- 現在の公開版は主に Windows デスクトップ環境向けです
- セキュリティ設定が厳しい場合、初回起動時に実行確認が必要になることがあります

## よくある質問

### インストーラー版とポータブル版の違いは？

- インストーラー版は長期的な常用に向いています
- ポータブル版は素早い体験や単体配布に向いています

### 自分でパスワードを設定しなくても使えますか？

はい。`随机生成` をクリックすると強力なパスワードが生成され、デスクトップに txt バックアップも保存されます。

### 何度もランダム生成すると以前のパスワードは消えますか？

いいえ。新しい記録は同じバックアップファイルに追記され、古い記録は残ります。

### パスワードを忘れたらどうなりますか？

パスワードは暗号化データを復元するために不可欠です。使い始める前に管理方法を決めておき、ランダムパスワードを使う場合はデスクトップ上のバックアップファイルを必ず保存してください。

### 最初のテストで重要なデータを使ってもいいですか？

おすすめしません。まずはテスト用フォルダーで流れを確認してから本番データを扱ってください。
