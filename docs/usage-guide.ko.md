# Usage Guide

## 대상 사용자

이 가이드는 `Folder Encryptor V5`를 처음 사용하는 사용자를 위한 문서입니다.

빠르게 훑어보고 싶다면 README부터 읽으면 되고, 본격적으로 사용할 계획이라면 이 페이지 전체를 읽는 것을 권장합니다.

## 제공 버전

현재 프로젝트는 두 가지 버전을 제공합니다.

- 설치형 버전: `dist/FolderEncryptor_Dino_SmartSetup.exe`
- 포터블 버전: `dist/FolderEncryptor_V5_Dino.exe`

일반적으로는 설치형 버전을 추천합니다. 빠르게 체험만 하고 싶거나 설치를 원하지 않으면 포터블 버전을 바로 사용할 수 있습니다.

## 화면 구성

메인 화면의 주요 항목:

- `Target Folder / 目标文件夹`: 암호화 또는 복호화할 폴더 선택
- `Browse`: 폴더 선택 창 열기
- `Master Password / 安全密码`: 직접 비밀번호 입력
- `随机生成`: 강력한 비밀번호 자동 생성
- `隐形加密 (Encrypt)`: 암호화 실행
- `一键还原 (Decrypt)`: 복호화 실행

## 랜덤 비밀번호 생성 기능

직접 비밀번호를 설정하고 싶지 않다면 `随机生成` 버튼을 클릭하면 됩니다.

프로그램은 강력한 비밀번호를 자동 생성하고, 비밀번호 분실을 방지하기 위해 그 내용을 데스크톱의 백업용 텍스트 파일에도 저장합니다.

이 백업 파일의 특징:

- 데스크톱에 txt 백업 파일이 생성됨
- `随机生成`을 누를 때마다 새 비밀번호 기록이 계속 추가됨
- 이전 기록은 덮어쓰지 않음

예시:

```text
【V5 专属加密密码备份】
密码: euZi^oahxuKq2Zm6

【V5 专属加密密码备份】
密码: JSE7CWTg54#CSH!Q
```

작업이 끝난 뒤에는 이 백업 파일을 더 안전한 위치로 옮겨 보관하는 것이 좋습니다.

## 시작 전 권장 사항

처음 사용하기 전에 중요하지 않은 파일 몇 개가 들어 있는 테스트 폴더를 준비해서 암호화와 복호화 과정을 한 번 끝까지 실행해 보세요.

이렇게 하면:

- 화면과 작업 순서에 익숙해질 수 있고
- 현재 컴퓨터 환경에서 정상 작동하는지 확인할 수 있으며
- 첫 사용부터 중요한 데이터를 바로 다루는 위험을 줄일 수 있습니다

## 폴더 암호화

권장 절차:

1. 프로그램 실행
2. `Target Folder / 目标文件夹` 에서 암호화할 폴더 선택
3. `Master Password / 安全密码` 에 비밀번호를 입력하거나 `随机生成` 클릭
4. 선택한 경로가 맞는지 확인
5. `隐形加密 (Encrypt)` 클릭
6. 처리 완료까지 대기
7. 결과 폴더를 열어 수동으로 확인

## 폴더 복호화

권장 절차:

1. 프로그램 실행
2. 이 도구로 처리한 폴더 선택
3. 원래 암호화 비밀번호 입력
4. `一键还原 (Decrypt)` 클릭
5. 복원된 파일이 정상적으로 열리는지 확인

## 주의 사항

### 1. 비밀번호 관리

- 비밀번호는 파일 복원의 핵심입니다
- 반드시 안전하게 보관하세요
- 랜덤 비밀번호를 사용했다면 데스크톱 백업 파일도 함께 보관하세요

### 2. 작업 전 백업

- 중요한 데이터를 암호화하기 전에 추가 백업을 권장합니다
- 특히 처음 실사용할 때는 유일한 원본을 바로 처리하지 않는 것이 좋습니다

### 3. 먼저 작은 규모로 테스트

- 먼저 작은 폴더 1~2개로 테스트하세요
- 작업 흐름이 안정적인 것을 확인한 뒤 중요한 데이터로 넘어가세요

### 4. 환경 관련 안내

- 현재 공개 버전은 주로 Windows 데스크톱 환경을 대상으로 합니다
- 보안 설정이 엄격한 시스템에서는 처음 실행할 때 수동 확인이 필요할 수 있습니다

## 자주 묻는 질문

### 설치형과 포터블 버전의 차이는 무엇인가요?

- 설치형은 장기적인 일반 사용에 더 적합합니다
- 포터블은 빠른 체험과 단일 파일 배포에 더 적합합니다

### 직접 비밀번호를 설정하지 않아도 되나요?

네. `随机生成`을 클릭하면 강력한 비밀번호를 자동 생성하고 데스크톱에 txt 백업도 저장합니다.

### 여러 번 랜덤 생성하면 예전 비밀번호가 덮어써지나요?

아니요. 새 비밀번호는 같은 백업 파일에 계속 추가되고, 이전 기록은 그대로 남습니다.

### 비밀번호를 잊어버리면 어떻게 되나요?

비밀번호는 암호화된 데이터를 복원하는 데 꼭 필요합니다. 사용 전에 비밀번호 관리 방법을 정해 두고, 랜덤 비밀번호를 사용했다면 데스크톱 백업 파일을 꼭 보관하세요.

### 처음 테스트할 때 중요한 데이터를 바로 써도 되나요?

권장하지 않습니다. 먼저 테스트용 폴더로 작업 흐름을 검증한 뒤 실제 데이터에 사용하는 것이 좋습니다.
