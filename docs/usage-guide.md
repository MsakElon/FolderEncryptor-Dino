# Usage Guide

## 适用对象

这份说明适合第一次接触 `Folder Encryptor V5` 的用户。

如果你只想快速上手，可以先看 README；如果你准备正式使用，建议完整读完本页。

## 运行方式

项目当前提供两种使用方式：

- 安装版：`dist/FolderEncryptor_Dino_SmartSetup.exe`
- 便携版：`dist/FolderEncryptor_V5_Dino.exe`

一般建议优先使用安装版；如果只是临时体验或不想安装，可以直接运行便携版。

## 界面说明

主界面中的关键区域包括：

- `Target Folder / 目标文件夹`：选择需要加密或解密的文件夹
- `Browse`：打开文件夹选择器
- `Master Password / 安全密码`：输入你自己的密码
- `随机生成`：自动生成强密码
- `隐形加密 (Encrypt)`：执行加密
- `一键还原 (Decrypt)`：执行解密

## 什么是隐形加密

`Folder Encryptor V5` 的一个特点是“隐形加密”。

这里的“隐形”主要体现在：

- 加密后文件名和目录结构通常不会明显变化
- 从外观看，文件夹还像原来的文件夹
- 但文件内容已经变成密文
- 如果没有正确密码直接打开，文件通常会无法打开，或者显示乱码、损坏、无法识别

这类方式比较适合低调保护资料内容，而不是把文件直接改成非常显眼的“已加密包”。

## 技术原理简述

从技术上看，它不是只对文字做简单替换，而是对文件内容整体做加密处理。

大致流程如下：

1. 递归扫描目标文件夹中的文件
2. 将每个文件按完整二进制数据读取
3. 使用密码和随机 `salt`，通过 `PBKDF2-HMAC-SHA256` 派生密钥
4. 使用 `cryptography.fernet.Fernet` 对文件内容进行加密和校验保护
5. 将加密后的数据写回原文件位置

因此它保护的是文件内容本身，而不只是文本文件里的文字。

## 随机生成密码功能

如果你不想手动设置密码，可以直接点击 `随机生成` 按钮。

程序会自动生成一组强密码，并把结果额外保存到桌面上的文本备份文件中，防止忘记密码。

这个备份文件的特点是：

- 会在桌面生成一份 txt 备份
- 每次点击 `随机生成` 都会继续追加新的密码记录
- 不会覆盖之前已经生成过的密码

示例格式如下：

```text
【V5 专属加密密码备份】
密码: euZi^oahxuKq2Zm6

【V5 专属加密密码备份】
密码: JSE7CWTg54#CSH!Q
```

建议在完成操作后把这个备份文件移动到更安全的位置，避免长期留在桌面。

## 开始前建议

在第一次使用前，建议先准备一份测试文件夹，里面放几份不重要的文档或图片，先完整跑通一次加密和解密流程。

这样做的好处是：

- 可以先熟悉界面和操作顺序
- 可以确认当前电脑环境运行正常
- 可以避免第一次就直接处理重要资料

## 文件夹加密

推荐操作步骤：

1. 启动程序
2. 在 `Target Folder / 目标文件夹` 中选择要加密的目标文件夹
3. 在 `Master Password / 安全密码` 中手动输入密码，或者点击 `随机生成`
4. 检查目标路径是否正确
5. 点击 `隐形加密 (Encrypt)`
6. 等待程序处理完成
7. 打开结果目录做一次人工确认

## 文件夹解密

推荐操作步骤：

1. 启动程序
2. 选择已被该工具处理过的文件夹
3. 输入原始加密密码
4. 点击 `一键还原 (Decrypt)`
5. 检查恢复后的文件能否正常打开

## 使用中的注意事项

### 1. 密码管理

- 密码是恢复文件的关键
- 请务必自行保存好密码
- 如果使用随机密码，请同时保管好桌面生成的备份文件

### 2. 先备份再操作

- 对重要资料进行加密前，建议额外备份一次
- 尤其是第一次正式使用时，不建议直接处理唯一原件

### 3. 先小规模测试

- 先拿一两个小文件夹测试最稳妥
- 确认流程稳定后，再处理更重要的数据

### 4. 环境建议

- 当前公开版本面向 Windows 桌面环境
- 如果系统安全策略较严格，首次运行可能需要确认是否允许执行

## 常见问题

### 安装版和便携版有什么区别？

- 安装版更适合作为常用工具长期保留
- 便携版更适合快速体验和单文件分发

### 不自己设密码可以吗？

可以。你可以直接点击 `随机生成`，程序会帮你生成强密码，并自动在桌面保存一份 txt 备份。

### 多次点击随机生成会不会覆盖之前的密码？

不会。新的密码会继续追加到同一个备份文本中，旧记录会保留。

### 忘记密码怎么办？

密码是加密数据恢复的重要凭据。建议在开始使用前，就确定好自己的密码管理方式；如果用了随机密码，也请及时保存桌面备份文件。

### 可以直接拿重要资料做第一次测试吗？

不建议。推荐先用测试文件夹验证流程，再处理正式数据。

## 推荐使用场景

- 个人隐私资料整理
- 课程作业和学习资料归档
- 本地项目文档的额外保护
- 需要临时打包保护的一组文件夹
